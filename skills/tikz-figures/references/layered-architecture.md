> **何时加载**：用户提及"系统架构""分层架构""端边云""四层架构""N层堆叠"，或文案描述自下而上的多层系统结构时加载。

## 分层系统架构图专项规则

分层系统架构图用于展示端-边-云-链、硬件-中间件-应用等多层系统结构，是论文中最常见的图表类型之一。

### 适用场景

用户提及"系统架构""分层架构""端边云""四层架构""N层堆叠"，或文案描述的是自下而上的多层系统结构。

### 布局核心原则

**每层内部结构**：不要只放3个平铺节点——每层应有主模块（`base_box`）+ 子组件（`small_box`），主模块在上，子组件在下方排列。例如"联邦学习协调器"（主）下挂"参数服务器""GPU Worker Pool"（子）。这样才能体现层内的复杂性。

**层间距**：相邻 zone 之间留 1.5–2.0cm 间隙，用于放置协议标签（MQTT/gRPC/RESTful）和跨层连线的弯折中转。

**zone 对齐**（关键教训）：所有 zone 必须左右对齐，用隐形锚点方案：
```latex
\node[inner sep=0pt, minimum size=0pt] (anchor_L) at (-2.5, 0) {};
\node[inner sep=0pt, minimum size=0pt] (anchor_R) at (16.5, 0) {};
% 每个zone的fit中加入：
fit=(anchor_L |- 底部节点.south)(anchor_R |- 顶部节点.north)(其他节点...)
```

**层标题胶囊**：用绝对 x 坐标（如 `at (-4.5, y)`），y 取该层中心。不要用 `$(zone.west)+(-2,0)$`。

### 跨层连线规则（反复踩坑总结）

1. **上行数据流线不穿越中间层节点**：从底层 SDK 出来的竖直线，必须检查路径上方每一层是否有节点。出发点的 x 坐标应选在两个节点列之间的空隙。例如三列节点在 x=0, 6, 12 时，上行线应走 x=3 或 x=9 的空隙
2. **多条上行线间距 ≥ 3cm**：从同一横跨节点（如 SDK）出发的多条上行线，出发 x 坐标间距至少 3cm，避免视觉上平行重叠
3. **协议标签放在层间空隙的右侧空白处**：不要放在上行线路径上，会被连线遮挡
4. **双向通路（如模型下发/梯度回传）走独立 rail**：在所有内容区域的最左侧或最右侧，距 zone 边界 ≥ 1.5cm
5. **反向连线（如 Oracle 预言机、事件回调）走 zone 外侧**：从源节点侧面出框，沿 zone 外边界走 rail，不穿越任何 zone。优先走左侧 rail 给 Oracle 预言机，右侧 rail 给事件回调，左右分开避免交叉
6. **短距离弯折走空白侧**：源和目标斜向相邻时，先水平走到空白区域再 `|-` 弯折进入目标，不要直接从 `.south` 弯折——会紧贴其他节点
7. **所有跨层连线加 `rounded corners=6pt`**：纯直角弯折在架构图中显得死板，圆角弯折更自然美观

### 数据库节点（cylinder）

用 `cylinder` 形状（`shape border rotate=90, aspect=0.25`）区分存储类组件（HDFS/Redis/缓存），比矩形节点更有辨识度。

**cylinder 布局规则**（关键教训）：
- cylinder 节点必须**与同层其他模块在相同 y 基线上**，不要为了"看起来不同"就偏移 y 坐标——这会导致层归属的视觉混淆（cylinder 看起来属于相邻层）
- 如果 cylinder 因为高度较大与相邻模块拥挤，增大该层的 zone 高度，而非移动 cylinder 的 y 坐标
- cylinder 的 `.west`/`.east` 锚点在圆弧上，连线入口方向用 `.north`/`.south` 更自然

### 同 x 坐标跨层直连规则

当源节点和目标节点在相同 x 坐标上、且属于相邻层时：
- **允许**直接垂直连接（`.north` → `.south`），不需要走外侧 rail——这是最自然、最简洁的连线方式
- **但连线必须不穿越中间的 zone 虚线框体**：如果两个 zone 之间有间隙（推荐 1.5-2.0cm），垂直直连自然经过间隙，是安全的
- **跨越 2 层以上**（中间隔了至少 1 个 zone）的同 x 连线：禁止直连，必须走外侧 rail

### 安全域/信任域绘制规范

当架构图需要表示安全域或信任域（虚线大框包裹多个 zone）时：
- 安全域框使用比 zone 更粗的虚线（`line width=1.5pt`）和更大的 `rounded corners=12pt`
- 安全域标题位置：放在框的**上方外侧**（`anchor=south west` at 框.north west + (0.5, 0.2)），不要放在左侧——左侧已有层标题胶囊
- 安全域标题与层标题的视觉区分：安全域标题用**不同颜色和形状**（如红色虚线框标题 vs 蓝色实心胶囊层标题），确保读者不混淆
- 安全域框与内部 zone 之间间距 ≥ 0.5cm
- 多个安全域之间的间隙标注（如"安全通道"）：用一条**水平虚线横贯间隙**+ 居中标签，而非孤立标签浮在空白中

### rail 标签边界距离

所有 rail 上的标签（如"模型更新""审计回调"）必须距图边界 ≥ 2.0cm。在设定 rail x 坐标时，同时计算标签宽度（中文约 0.4cm/字），确保标签不贴边或被裁切。如果 rail 在右侧且标签较长，增大 `border` 值（如从 25pt 增到 35pt）。

### 自检清单（分层架构图专用）

- [ ] 所有 zone 左右边界对齐（用隐形锚点）
- [ ] 层标题胶囊 x 坐标统一
- [ ] **层标题与安全域标题视觉可区分**（不同颜色/形状/位置）
- [ ] 上行数据流线不穿越任何节点框
- [ ] 多条上行线间距 ≥ 3cm
- [ ] 反向连线走 zone 外侧 rail，不穿越 zone
- [ ] 每条连线的弯折处在空白区域，不紧贴节点边缘
- [ ] 最后进入目标节点的一段是纯水平或纯垂直（用 `|-` 或 `-|` 保证）
- [ ] 协议标签不被连线遮挡
- [ ] cylinder 节点与同层模块在相同 y 基线上
- [ ] **rail 标签距图边界 ≥ 2.0cm**
- [ ] **连线数量与画图指令中列出的一致**（逐条核对）
- [ ] 安全域间隙有水平虚线标注，标签不孤立浮空
- [ ] **渲染后视觉审查**（必须打开 PNG 查看）：
  - [ ] 跨层连线在白色/浅色背景上清晰可辨（避免浅橙/浅黄线在白色背景上消失）
  - [ ] **所有连线无斜线段**——全部由水平段+垂直段组成（L 型弯折），无从 A 斜穿到 B 的情况
  - [ ] **弯折点远离节点边缘**（≥ 0.5cm），弯折后的线段不穿过其他节点
  - [ ] 左右 rail 上的文字标签可读、不被裁切
  - [ ] 安全域标题与层标题视觉上能一眼区分
  - [ ] 各层模块数量均衡，无某层过于空旷
  - [ ] 整图纵横比协调，嵌入论文不会被过度压缩
